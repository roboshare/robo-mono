# v0.3.2-internal.3

## Scope

Internal testnet patch cut from the current `dev` head. This release includes
the Robomata server-side Sui facility-assignment persistence fix required before
promoting the current testnet web deployment to production.

## Included Work

- Patches Postgres Sui facility assignment persistence to update only the
  assignment-related submission fields instead of writing a stale full payload.
- Preserves the live evidence root check before assigning an onchain Sui
  facility to a submission.
- Removes successful assignment reservation fields from the persisted evidence
  commit state so subsequent evidence commits can proceed normally.

## Deployment Notes

- Deploy the web app from `main` after the release PR merges.
- Keep the existing `ROBOMATA_SUI_PACKAGE_ID`; no Sui package republish is
  needed for this patch.
- No subgraph redeploy is required for this patch.
