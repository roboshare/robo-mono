# v0.3.0-internal.1

## Scope

First internal cut of the `v0.3.0` testnet release line from `release/testnet-v0.3.0`.

This release opens the next testnet stabilization line after the `v0.2.x` internal rollout. It packages the position-manager architecture, release-branch governance updates, indexing hardening, market-flow fixes, protocol configuration hardening, and Robomata evidence-rail work that landed on `dev`, then uses the release branch to validate fresh Sepolia and Polygon Amoy deployments before any beta or public testnet tag.

## Included Work

- Position ownership, transfer, redemption, settlement, and marketplace state now route through `PositionManager`
- Settlement claim accounting is centralized through the registry router path
- Earnings, transfer, and terminal-settlement flows preserve seller and holder accounting across manager-backed positions
- Marketplace and treasury flows include additional guardrails around manager wiring, listing locks, transferred credits, and early-settlement funding
- Subgraph indexing was hardened around registration metadata, manager-backed consumer events, generated manifests, ABI/codegen/build preparation, and slimmed market reads
- Web markets and partner flows were updated for manager-backed reads, active-chain smart-wallet writes, active-chain receipt validation, settlement claims, metadata retries, APR calculations, event-history guards, payout display, and card layout stability
- Web protocol configuration now separates deployment metadata, token units, default chain selection, and chain-aware buffer calculations so release previews can target configured testnets without hard-coded local assumptions
- Markets pages now reduce redundant RPC/subgraph load by relying on release-aware protocol configuration and narrower refresh behavior
- Robomata FacilitySubmission documentation and Sui evidence rails are included for the parallel Robomata testnet workflow, including a Sui testnet deploy script and README updates
- Review-finding workflow guidance is documented for repo-level release and PR follow-up
- Release governance now reflects the current `dev -> release/* -> main` branch model and avoids creating new long-lived `integration/*` branches

## Deployment Model

- Integration happens on `dev`
- Release stabilization happens on `release/testnet-v0.3.0`
- Internal release documents and tags use the launch-stage format, starting with `v0.3.0-internal.1`
- Vercel production remains pinned to `main`
- Internal validation should use release-branch preview deployments
- Final public testnet ship should happen by merging the approved release branch into `main`

## Deployment Expectations

- Fresh protocol deployments are expected as part of this release line for:
  - `Sepolia`
  - `Polygon Amoy`
- Deployment artifacts, frontend contract metadata, subgraph network config, and managed subgraph endpoints should be recorded before promoting this release beyond internal validation
- `Sepolia` remains the default reference network for product QA and tester guidance unless release validation explicitly changes that default

## Validation Expectations

- `yarn evm:test` passes against the manager-backed protocol model
- `yarn web:check-types` passes with the release-branch frontend contract and network configuration
- `yarn web:lint` passes with the release-branch frontend and config updates
- `yarn deploy:verify --network sepolia --keystore <keystore>` completes successfully and records Sepolia deployment outputs
- `yarn deploy:verify --network polygonAmoy --keystore <keystore>` completes successfully and records Polygon Amoy deployment outputs
- `yarn subgraph:build --network sepolia` completes successfully from a fresh worktree state
- `yarn subgraph:build --network polygon-amoy` completes successfully from a fresh worktree state
- Managed subgraph endpoints for Sepolia and Polygon Amoy are wired through `NEXT_PUBLIC_SUBGRAPH_URL_11155111` and `NEXT_PUBLIC_SUBGRAPH_URL_80002`
- Core investor and partner flows work end to end on Sepolia before expanding tester scope to Polygon Amoy
