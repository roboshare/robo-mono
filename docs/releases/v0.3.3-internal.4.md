# v0.3.3-internal.4

## Scope

Internal testnet patch cut from the current `dev` head after
`v0.3.2-internal.3`. This release packages the EVM, subgraph, and testnet web
surface updates that need to be auditable independently from the Robomata
Overflow product release line.

## Included Work

- Adds PartnerManager indexing to the hosted subgraph, including authorized,
  revoked, and renamed partner state.
- Backs the Robomata operator gate with subgraph authorization data so app
  access checks do not rely only on browser contract reads.
- Preserves the latest dev-side workflow token-permission fix for testnet CI
  coverage while keeping the matching main-side hardening commit in history.
- Carries forward the RPC-load reductions, app-host navigation hardening, and
  production host fallback fixes needed for public testnet surfaces.
- Includes the current Robomata Sui, Walrus, facility monitoring, rental, and
  agent-supervision code paths behind their configured environment gates.

## Deployment Notes

- Redeploy hosted subgraphs for every active public testnet environment after
  this release branch merges:
  - `sepolia`
  - `polygon-amoy`
  - `arbitrum-sepolia`
  - `monad-testnet`
- Confirm each deployment has a `PartnerManager` address and start block before
  publishing the new subgraph version.
- Revalidate `/markets`, `/operator`, and `/robomata/submissions` against the
  redeployed subgraphs before promoting the production web deployment.
- Keep this tag separate from the Robomata Overflow product tag so EVM/subgraph
  rollout state remains independently traceable.
